3. Revisión

Enviadas {{ pr.stats.linesSubmitted }} Indexarían {{ pr.stats.ok }} Ignoradas {{ pr.stats.skipLine }} Sin celda {{ pr.stats.noCell }} No id válido {{ pr.stats.invalidId }}

Las columnas Cómo se leyó y Celda tomada muestran qué campo se usa antes de normalizar. El volcado vuelve a leer el archivo con el mismo perfil: lo que marques en Vista previa con «Aplicar» sirve para comprobar reglas y ver la fila en verde, pero no sustituye líneas en el fichero. Para que el volcado coincida, ajusta el perfil (en una celda ya se separan solas las URL con campos tras : o con /login) o exporta una lista ya limpia. Las filas OK con URL o «toda la línea (modo plano)» también muestran reparación por si el automático falló.

Reparación manual (filas en duda):
  1. En Vista previa, edita el cuadro de texto: deja solo lo que quieres indexar (p. ej. el RUT 174443602 o con DV 17.444.360-2, o un correo válido).
  2. Pulsa Aplicar. Si las reglas activas lo reconocen, la fila pasa a OK.
  3. Restaurar vuelve al valor detectado por el perfil. Las URLs largas se pueden pegar y recortar en el cuadro; no hace falta reescribir toda la línea fuera de la app.
  4. Si ves segmentos numerados (ruta, campos con «:»), pulsa uno para copiarlo al cuadro y Aplicar (útil si el automático eligió mal el trozo). La ruta y la contraseña no se indexan.
